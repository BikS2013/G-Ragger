# GeminiRAG - Refined Request Specification

## 1. Summary

GeminiRAG is a TypeScript CLI tool that provides workspace-based document management and semantic search powered by the Google Gemini File Search API. Users can create named workspaces (backed by Gemini File Search stores), upload content from multiple sources (local files, web pages, YouTube videos, and personal notes), and query those workspaces using natural language with optional metadata filtering. Each upload carries auto-generated metadata (timestamp, title) and supports user-managed properties (expiration date, status flags). The tool maintains a local registry to track upload metadata that extends beyond what the Gemini API stores natively.

---

## 2. Functional Requirements

### Workspace Management

- **FR-01**: The tool MUST allow the user to create a new workspace by providing a workspace name.
- **FR-02**: The tool MUST create a corresponding Gemini File Search store for each workspace.
- **FR-03**: The tool MUST allow the user to list all existing workspaces.
- **FR-04**: The tool MUST allow the user to delete a workspace (and its backing Gemini File Search store and all associated uploads).
- **FR-05**: The tool MUST allow the user to view details of a specific workspace (name, creation date, number of uploads, metadata labels in use).

### Content Upload - General

- **FR-06**: Every upload MUST be automatically assigned a UTC timestamp at the moment of upload.
- **FR-07**: Every upload MUST be automatically assigned a title by the tool (source-specific; see Section 3).
- **FR-08**: The user MUST NOT be required to provide timestamps or titles; the tool generates them.
- **FR-09**: Every upload MUST be indexed in the Gemini File Search store for semantic search.
- **FR-10**: The tool MUST store upload metadata (timestamp, title, source type, expiration date, flags, custom metadata) in a local registry file, since the Gemini API's custom metadata is limited to key-value pairs set at import time.

### Content Upload - Disk File

- **FR-11**: The tool MUST accept a local file path and upload the file to the workspace's Gemini File Search store.
- **FR-12**: The auto-generated title for disk files MUST be the file name (without the directory path).
- **FR-13**: The tool MUST validate that the file exists and that its MIME type is supported by the Gemini File Search API before uploading.

### Content Upload - Web Page

- **FR-14**: The tool MUST accept a URL, fetch the web page content, convert it to a text-based format (e.g., markdown or plain text), and upload the resulting content to the workspace.
- **FR-15**: The auto-generated title for web pages MUST be extracted from the page's HTML `<title>` element. If no title is found, the tool MUST use the URL's hostname + path as the title.
- **FR-16**: The tool MUST store the source URL as metadata on the upload.

### Content Upload - YouTube Video

- **FR-17**: The tool MUST accept a YouTube video URL, extract the video's transcript/captions, and upload the transcript text to the workspace.
- **FR-18**: The auto-generated title for YouTube videos MUST be the video's title as retrieved from YouTube.
- **FR-19**: The tool MUST store the YouTube video URL as metadata on the upload.
- **FR-20**: If a transcript is not available for the video, the tool MUST report an error and NOT create a partial upload.

### Content Upload - Personal Note

- **FR-21**: The tool MUST accept free-text input as a personal note and upload it to the workspace.
- **FR-22**: The auto-generated title for personal notes MUST be generated by the tool based on the note's content (e.g., first N characters, or an AI-generated summary title).
- **FR-23**: Personal notes MUST be stored as plain text files in the Gemini File Search store.

### Upload Metadata Management

- **FR-24**: The tool MUST allow the user to change the title of any upload in a workspace.
- **FR-25**: The tool MUST allow the user to remove (delete) an upload from a workspace, removing both the Gemini File Search document and the local registry entry.
- **FR-26**: The tool MUST allow the user to set an expiration date on an upload (optional; ISO 8601 date format).
- **FR-27**: The tool MUST allow the user to clear an expiration date from an upload.
- **FR-28**: The tool MUST allow the user to set one or more status flags on an upload. Supported flags: `completed`, `urgent`, `inactive`.
- **FR-29**: The tool MUST allow the user to clear flags from an upload.
- **FR-30**: The tool MUST display expiration warnings when listing uploads that have expired or are expiring within a configurable warning window.

### Metadata Labels

- **FR-31**: The tool MUST provide a command to list all distinct metadata labels (keys) currently in use across uploads within a specified workspace.
- **FR-32**: The metadata labels include both system-managed labels (timestamp, title, source_type, source_url, expiration_date, flags) and any future user-defined labels.

### Querying

- **FR-33**: The tool MUST allow the user to ask a natural language question against a workspace.
- **FR-34**: The query MUST be routed through the Gemini `generateContent` API with the workspace's File Search store as the tool.
- **FR-35**: The tool MUST support metadata filters when querying (e.g., filter by source_type, flags, date ranges, or any metadata key-value pair).
- **FR-36**: The tool MUST display the answer along with citations (source document titles and relevant text excerpts) from the grounding metadata.
- **FR-37**: The tool MUST support querying across multiple workspaces in a single query.

### Upload Listing

- **FR-38**: The tool MUST allow the user to list all uploads within a workspace, showing: title, source type, timestamp, expiration date (if set), and active flags.
- **FR-39**: The listing MUST support filtering by metadata (source type, flags, expiration status).
- **FR-40**: The listing MUST support sorting by timestamp (ascending/descending).

## 3. Content Sources - Detailed Behavior

| Source Type   | Input                     | Title Generation                                   | Stored Metadata                              | Upload Format        |
|---------------|---------------------------|------------------------------------------------------|----------------------------------------------|----------------------|
| Disk File     | Local file path           | File name (basename without directory)               | `source_type: file`, original file path      | Original file        |
| Web Page      | URL                       | HTML `<title>` tag; fallback: hostname + path        | `source_type: web`, `source_url: <url>`      | Extracted markdown/text |
| YouTube Video | YouTube URL               | Video title from YouTube                             | `source_type: youtube`, `source_url: <url>`  | Transcript text      |
| Personal Note | Free-text (stdin or arg)  | AI-generated or first 60 chars + ellipsis            | `source_type: note`                          | Plain text           |

## 4. Upload Metadata - Field Definitions

| Field            | Type        | Auto-generated | Mutable | Required | Description                                                    |
|------------------|-------------|----------------|---------|----------|----------------------------------------------------------------|
| `id`             | string      | Yes            | No      | Yes      | Unique identifier for the upload (UUID)                        |
| `document_name`  | string      | Yes            | No      | Yes      | Gemini File Search document resource name                      |
| `title`          | string      | Yes            | Yes     | Yes      | Human-readable title (auto-generated, user-editable)           |
| `timestamp`      | string      | Yes            | No      | Yes      | ISO 8601 UTC datetime of upload creation                       |
| `source_type`    | enum        | Yes            | No      | Yes      | One of: `file`, `web`, `youtube`, `note`                       |
| `source_url`     | string      | Yes            | No      | No       | Original URL for web/youtube sources; file path for disk files |
| `expiration_date`| string/null | No             | Yes     | No       | ISO 8601 date; null if no expiration                           |
| `flags`          | string[]    | No             | Yes     | No       | Array of status flags: `completed`, `urgent`, `inactive`       |

## 5. Metadata Operations

| Operation            | Command Example                                      | Description                                                  |
|----------------------|------------------------------------------------------|--------------------------------------------------------------|
| Change title         | `geminirag update-title <workspace> <upload-id> "New Title"` | Updates the title of a specific upload                   |
| Remove upload        | `geminirag remove <workspace> <upload-id>`           | Deletes the Gemini document and local registry entry         |
| Set expiration       | `geminirag set-expiration <workspace> <upload-id> 2026-06-01` | Sets/updates the expiration date                      |
| Clear expiration     | `geminirag clear-expiration <workspace> <upload-id>` | Removes the expiration date                                  |
| Set flags            | `geminirag flag <workspace> <upload-id> --add urgent completed` | Adds one or more flags to an upload                |
| Clear flags          | `geminirag flag <workspace> <upload-id> --remove urgent`       | Removes one or more flags from an upload           |
| List metadata labels | `geminirag labels <workspace>`                       | Lists all distinct metadata keys used in the workspace       |

## 6. Workspace Operations

| Operation                   | Command Example                                             | Description                                                     |
|-----------------------------|-------------------------------------------------------------|-----------------------------------------------------------------|
| Create workspace            | `geminirag create <name>`                                   | Creates a new workspace + Gemini File Search store              |
| List workspaces             | `geminirag list`                                            | Lists all workspaces with summary info                          |
| Delete workspace            | `geminirag delete <name>`                                   | Deletes workspace, its Gemini store, and all local data         |
| View workspace details      | `geminirag info <name>`                                     | Shows workspace metadata, upload count, labels in use           |
| Upload disk file            | `geminirag upload <workspace> --file <path>`                | Uploads a local file to the workspace                           |
| Upload web page             | `geminirag upload <workspace> --url <url>`                  | Fetches and uploads web page content                            |
| Upload YouTube video        | `geminirag upload <workspace> --youtube <url>`              | Extracts and uploads YouTube transcript                         |
| Add personal note           | `geminirag upload <workspace> --note "text content"`        | Uploads a personal note                                         |
| List uploads                | `geminirag uploads <workspace> [--filter ...] [--sort ...]` | Lists uploads with optional filters and sorting                 |
| Query workspace             | `geminirag ask <workspace> "question" [--filter ...]`       | Queries the workspace with optional metadata filters            |
| Query multiple workspaces   | `geminirag ask <w1> <w2> "question" [--filter ...]`         | Queries across multiple workspaces                              |

## 7. Configuration Requirements

| Variable          | Purpose                                             | Source                          | Required | Expirable |
|-------------------|-----------------------------------------------------|---------------------------------|----------|-----------|
| `GEMINI_API_KEY`  | Authentication for Google Gemini API                | Google AI Studio / Cloud Console| Yes      | Yes       |
| `GEMINI_MODEL`    | Model to use for queries (e.g., `gemini-2.5-flash-lite`) | Config file or env var     | Yes      | No        |

### Configuration Rules

- Configuration MUST be loaded from environment variables or a config file (e.g., `~/.geminirag/config.json` or `.env`).
- Priority: environment variables > `.env` file > config file.
- If `GEMINI_API_KEY` is not provided, the tool MUST raise a clear error with instructions on how to obtain and set the key. No fallback or default values.
- If `GEMINI_MODEL` is not provided, the tool MUST raise a clear error. No fallback or default values.
- The config file SHOULD support a `GEMINI_API_KEY_EXPIRATION` field (ISO 8601 date) so the tool can warn the user when the API key is approaching expiration.
- The local registry of workspaces and uploads MUST be stored at `~/.geminirag/registry.json`.

## 8. Acceptance Criteria

### Workspace Management
- **AC-01**: Running `geminirag create my-research` creates a workspace and a corresponding Gemini File Search store. Running `geminirag list` shows the workspace.
- **AC-02**: Running `geminirag delete my-research` removes the workspace, its Gemini store, and all local registry data. Running `geminirag list` no longer shows it.
- **AC-03**: Running `geminirag info my-research` displays workspace name, creation date, upload count, and metadata labels in use.

### Content Upload
- **AC-04**: Running `geminirag upload my-research --file ./report.pdf` uploads the file; `geminirag uploads my-research` shows an entry with title "report.pdf", source_type "file", and a timestamp.
- **AC-05**: Running `geminirag upload my-research --url https://example.com/article` fetches the page, extracts the title from HTML, and uploads the content. The upload listing shows the extracted title and source_type "web".
- **AC-06**: Running `geminirag upload my-research --youtube https://youtube.com/watch?v=xyz` extracts the transcript, uses the video title, and uploads. The upload listing shows the video title and source_type "youtube".
- **AC-07**: Running `geminirag upload my-research --note "Meeting notes: discussed Q3 targets"` creates an upload with an auto-generated title derived from the note content.
- **AC-08**: Uploading a file with an unsupported MIME type produces a clear error message listing supported types.
- **AC-09**: Uploading a YouTube video without available transcripts produces a clear error message.

### Metadata Operations
- **AC-10**: Running `geminirag update-title my-research <id> "Better Title"` changes the title. Running `geminirag uploads my-research` shows the new title.
- **AC-11**: Running `geminirag remove my-research <id>` deletes the upload. It no longer appears in listings or Gemini search results.
- **AC-12**: Running `geminirag set-expiration my-research <id> 2026-05-01` sets the expiration. The upload listing shows the expiration date.
- **AC-13**: Running `geminirag flag my-research <id> --add urgent` adds the flag. The upload listing shows the "urgent" flag.
- **AC-14**: Running `geminirag labels my-research` lists all metadata keys (e.g., title, source_type, flags, expiration_date).
- **AC-15**: Expired uploads are visually marked in listings with an "[EXPIRED]" indicator.

### Querying
- **AC-16**: Running `geminirag ask my-research "What are the main findings?"` returns a natural language answer with citations referencing specific uploads.
- **AC-17**: Running `geminirag ask my-research "What are the main findings?" --filter source_type=web` returns results only from web page uploads.
- **AC-18**: Running `geminirag ask my-research my-other "Compare findings"` queries across both workspaces.

### Configuration
- **AC-19**: Running any command without `GEMINI_API_KEY` set produces an error: "GEMINI_API_KEY is required. Obtain it from https://aistudio.google.com/apikey".
- **AC-20**: Running any command without `GEMINI_MODEL` set produces an error: "GEMINI_MODEL is required. Set it in config or environment."
- **AC-21**: If `GEMINI_API_KEY_EXPIRATION` is set and the date is within 7 days, the tool prints a warning at startup.

## 9. Assumptions

1. **Gemini File Search API availability**: The Gemini File Search API (v1beta) is assumed to be stable enough for production use. The tool will use the `@google/genai` JavaScript/TypeScript SDK.
2. **YouTube transcript extraction**: A third-party library (e.g., `youtube-transcript` or `youtubei.js`) will be used to extract transcripts. This assumes publicly available captions exist on the target video.
3. **Web page content extraction**: A library like `mozilla/readability` or `turndown` will be used to convert HTML to clean text/markdown for upload.
4. **Metadata filter syntax**: The Gemini File Search metadata filter follows the AIP-160 filter syntax (e.g., `key=value`, `key>value`). The tool will construct these filters from user-provided CLI arguments.
5. **Local registry as metadata store**: Since the Gemini API's custom metadata is set at import time and is not easily updatable, mutable metadata (title changes, flags, expiration dates) will be managed in a local JSON registry. The Gemini store will hold a subset of metadata (source_type, source_url) as custom metadata for query-time filtering.
6. **Title generation for notes**: The tool will generate titles for personal notes using a simple heuristic (first 60 characters trimmed to word boundary + ellipsis), not AI-based generation, to avoid additional API costs. This can be revisited.
7. **Single-user tool**: The tool is designed for single-user local use. No multi-user access, authentication, or concurrency control is required.
8. **CLI-only interface**: The tool is a command-line interface only. No web UI or API server.
9. **File size limits**: The tool will respect Gemini API file size limits (currently 2GB per file via the Files API). Large files will produce a clear error.
10. **Workspace names are unique**: Each workspace must have a unique name locally. The tool will enforce this.

---

## 10. Out of Scope

1. **Web UI or API server**: The tool is CLI-only. No REST API, web dashboard, or Electron UI.
2. **Real-time sync**: No background processes to sync metadata between local registry and Gemini API.
3. **User authentication/authorization**: Single-user tool; no login or access control.
4. **File versioning**: No version history for uploads. Updating content requires removing and re-uploading.
5. **Bulk import**: No batch import command (e.g., import all files from a directory). Individual uploads only.
6. **Custom chunking configuration**: The tool will use Gemini's default chunking. Custom chunk sizes are not exposed to the user (can be added later).
7. **Scheduled expiration actions**: The tool displays expiration warnings but does NOT automatically delete expired uploads.
8. **Full-text search**: Only semantic search via Gemini is supported; no local keyword search.
9. **Non-text content**: Audio, video, and image files are not directly supported (only text-extractable content: documents, web pages, transcripts).
10. **Offline mode**: The tool requires internet access to communicate with the Gemini API. No offline querying.
11. **Export/backup**: No command to export workspace data to a local archive.
12. **User-defined custom metadata keys**: Only system-defined metadata fields are supported in v1. Arbitrary user-defined key-value pairs are out of scope.
